# Dula User Journey Map

## 1. Consumer Journey (P2P & Everyday Payments)
### Phase A: Awareness & Onboarding
1. Discovers Dula via social media.
2. Installs app or opens web onboarding page.
3. registers using thirdweb account abstraction frontend integration not backend as is is now, Provides phone, sets their username, gets a system generated number like binanceID public identifier and their AA wallet.
 
5. Links Mpesa wallet if the registereed number isnt mpesa registered otherwise the user can skip that part

### Phase B: Activation
1. Adds funds by depositing from external wallets.
2. Sends first P2P transfer to contact handle; sees instant receipt with masked identifiers.






## 2. Merchant Journey (SME / PoS)
### Phase A: Onboarding
1. registers as a merchant  using their consumer account which was already registered; provides business name.

2. Configures payout destinations: Mpesa Till/PayBill, phone, Arbitrum wallet.

### Phase B: Setup & Integration

3. Enables DeFi vault access (optional) and sets allocation percentage of idle balance.

### Phase C: Live Operations
1. Accepts payments via pochi-like flows, QR, or on-chain invoices.
2. Dashboard shows real-time ledger, pending settlements, and yield earnings.
3. Initiates off-ramps to Mpesa or bank; monitors notifications for reconciliation.
